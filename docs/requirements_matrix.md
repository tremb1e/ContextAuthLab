# Prompt Requirements Matrix

| ID | Area | Requirement | Acceptance |
|---|---|---|---|
| APP-01 | App shell | Kotlin + Jetpack Compose + Material 3, minSdk 26, targetSdk 35, ContextAuthLab branding | Debug APK builds and launches as ContextAuthLab |
| APP-02 | Consent | Significant disclosure before collection | User must consent before Home collection can start |
| APP-03 | Onboarding | Accessibility, battery optimization, notification permission flow | Automatic collection remains blocked until all three permissions pass |
| APP-04 | Home | `Collection Status` displays server connection state, automatic collection state, latest connectivity-test time, latest upload time, and latest server response; host-only server target, masked device ID, rule version, permissions, sensors, ClockSync, automatic collection readiness | No Home start/stop buttons; server connection chip triggers a fresh `/ready` readiness test; missing Accessibility, battery, and notification permissions are prompted one by one; no NTP host names in UI; collection starts automatically when required permissions, valid research `device_id`, and screen/unlock state pass; network, ClockSync, Wi-Fi, and rules refresh failures do not block local sampling |
| APP-05 | Researcher settings | Hidden version/Home server gesture, editable server URL, readiness test, reset, clear queue, export diagnostics | URL save/reset requires confirmation where applicable and triggers ClockSync/config/rule refresh; override banner appears when URL is changed |
| APP-06 | Built-in tasks | Eight C0-C7 adult-oriented task categories with sitting posture guidance and task labels | C1 scrollable protocol, C4 simulated phone settings, C5 fullscreen landscape blue-ball tapping challenge, C6 fullscreen local video player with overlaid pause/speed/seek/orientation controls, C7 wrist guide with stable forearm, wrist pivot, left-right swing, lateral translation, and forward-back flexion; BUILTIN_TASK batches include English task fields |
| APP-07 | Details | Runtime overview, app/device info, sensor rates, ScreenGate history, queue, upload, timing, local upload history, ClockSync | Details page has no one-click Diagnostics export button; local upload history shows file name, time, size, status, and response/error text |
| APP-08 | Device ID | HMAC-SHA256(serverStudySalt, ANDROID_ID), 64 lowercase hex, cached | No hardware IDs such as IMEI/serial/MAC/MediaDrm |
| APP-09 | Accessibility | Read-only node structure, foreground package/activity metadata, `viewIdResourceName`, visible non-editable component text, input-method-visible state, and broad UI component attributes | No performAction, gestures, screenshots, auto-click, raw input-field text, per-key timestamps, key intervals, key hold durations, touch positions, or touch trajectories |
| APP-10 | Sensor collection | Accelerometer, gyroscope, magnetic field at fixed 100 Hz | registerListener uses 10,000 us sampling period; status displays measured acc/gyro/mag rates |
| APP-11 | ScreenGate | Stop collection while screen off or locked | Existing redacted samples/events are flushed or queued before pause, then the same task context can resume after unlock |
| APP-12 | Redaction and UI semantics | Drop password nodes and editable text; upload plaintext foreground `app_package_name`, Activity/ComponentName, `viewIdResourceName`, and fixed-format-sensitive-redacted non-editable component text; apply server-provided text/content redaction rules when present | Server rule list is initialized from an editable JSON file; package-level app skipping is removed; Android keeps baseline sensitive-string redaction even when server rules are empty; serialized batches contain no raw input-field text; server schema rejects editable raw text and missing `diagnostics.redaction_applied: true` |
| APP-15 | Internationalization | Chinese and English UI content selected by system language | Major participant-facing UI, tasks, notification, protocol, details, and dialogs have Chinese/English variants |
| APP-13 | Transport | UTF-8 JSON -> LZ4 frame -> SHA-256 envelope over HTTP(S), with non-empty `session_id` for task and foreground collection sessions | No AES/PBKDF2/AAD/public-key envelope protocol |
| APP-14 | Failure queue and upload history | filesDir/upload_queue, metadata DB, 200 MB FIFO, retry/dead letter, local upload history | Offline batches are persisted and replayable; uploads/replays/queued failures are recorded for Details history |
| APP-16 | Default foreground collection | No third-party collection allow/deny switch; when consent and required permissions pass, foreground app UI, global timing-only touch events, and sensors are collected together | Sensor-only batches are retained; Accessibility UI events and global touch timing are included when present |
| SRV-01 | Server API | Pure FastAPI API: health/config/rules/ingest/metrics | Config exposes China-region ClockSync metadata; `/api/v1/rules` is initialized from `SERVER_RULES_FILE`; no templates/static/dashboard routes |
| SRV-02 | Ingest | Validate envelope, hash, LZ4, JSON, Pydantic schema | Valid sample batch returns stored=true |
| SRV-03 | Privacy checks | Validate schema-level privacy invariants after server secondary scan removal | Reject editable raw text, password-node presence, missing redaction marker, invalid task contract, or bad envelope without logging raw payload |
| SRV-04 | Storage | Store by device_id/date, meta, indexes, by_category, quarantine | Safe path resolution prevents traversal; repeated identical batch IDs are idempotent, conflicting duplicates are rejected without overwrite |
| SRV-05 | Logging/metrics | JSON lines logs and Prometheus metrics | No full device_id, raw payload, salt, key, or plain IP |
| DEV-01 | Docker | Server image plus Android APK artifact image; server runs non-root with bind-mounted data/logs | `docker build -t contextauthlab/server:latest ./server`, `make build-app-image` after the debug APK exists, and `tools/test_docker_deployment.sh` pass |
| DEV-02 | Tests | Android unit, APK build, server pytest, E2E, Docker smoke | Commands in README and Makefile pass in this environment |

Explicitly out of scope: authentication model, context routing model, MoE, training, inference, hidden collection, screenshots, automatic control, gRPC/proto, AES content encryption, anomaly detection, Dashboard/Web UI.
